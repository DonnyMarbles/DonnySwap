# LPVault

**Address**: [`0x4f86cc07AbAE6F4BD749857B2EC7C7e67C5AcAbf`](https://peaq.subscan.io/account/0x4f86cc07AbAE6F4BD749857B2EC7C7e67C5AcAbf?tab=contract)

Holds MRBL-PEAQ LP from DSFO minting (30% per mint) and FeeSplitter (30% of MRBL-PEAQ protocol fees). Provides time-weighted redemption for DSFO holders and distributes surplus via harvest.

**Inherits**: Ownable, ReentrancyGuard

**Source**: `contracts/LPVault.sol`

## Constants

| Name | Value | Description |
|------|-------|-------------|
| `BPS` | 10000 | Basis points denominator |
| `DAYS_365` | 365 days (31,536,000s) | Time weight maturity period |

|----------|------|-------------|
| `mrblPeaqLP` | IERC20 | The MRBL-PEAQ LP token |
| `dsfoToken` | address | DSFO contract (authorized caller for deposits and redemptions) |
| `feeSplitter` | address | FeeSplitter contract |
| `feeManager` | address | Receives DSFO bonus from harvest |
| `treasury` | address | Receives treasury share from harvest |

| `lastRedemptionTime` | mapping(address => uint256) | Per-holder cooldown timer |
| `redemptionCooldown` | uint256 | Min time between redemptions per address (default 48 hours) |
| `lastHarvestTime` | uint256 | Timestamp of last harvest |
| `harvestMinInterval` | uint256 | Min time between harvests (default 7 days) |
| `harvestBountyCap` | uint256 | Max caller bounty per harvest (default 0.01 ether) |
| `harvestBountyBps` | uint256 | Bounty percentage (default 100 = 1%) |

### Harvest Tier Configuration

Three tiers based on vault health, each must sum to 10000:

| Tier | Condition | Retain | DSFO Bonus | Treasury |
|------|-----------|--------|------------|----------|
| **Healthy** | Balance ≥ 80% of target | 6000 (60%) | 2500 (25%) | 1500 (15%) |
| **Medium** | Balance 50–80% of target | 8000 (80%) | 1200 (12%) | 800 (8%) |
| **Low** | Balance under 50% of target | 9500 (95%) | 300 (3%) | 200 (2%) |

## Deposits

### `depositFromMint(uint256 tokenId, uint256 lpAmount)` — onlyDsfoToken

Called by DSFO contract during mint. Records a MintRecord and increments `totalActiveMintCost`.

```solidity
function depositFromMint(uint256 tokenId, uint256 lpAmount) external onlyDsfoToken
```

**Reverts**:
- `lpAmount == 0`
- Token already has a mint record (`mintTimestamp != 0`)

### `depositFromFees(uint256 lpAmount)` — permissionless

Accept LP deposits from any source. Caller must have approved LP tokens.

```solidity
function depositFromFees(uint256 lpAmount) external
```

Used by FeeSplitter to deposit the 30% MRBL-PEAQ share. Can also be called by anyone wanting to donate LP to the vault.

## Redemption

### `processRedemption(uint256 tokenId, address holder) -> uint256` — onlyDsfoToken

Called by DSFO contract when a holder burns their NFT.

```solidity
function processRedemption(uint256 tokenId, address holder)
    external onlyDsfoToken nonReentrant returns (uint256 lpReturned)
```

**Calculation**:

```
Step 1 — Time Weight (linear 0% -> 80% over 365 days):
  daysSinceMint = block.timestamp - mintTimestamp
  timeWeightBps = min(8000, (8000 * daysSinceMint) / DAYS_365)
  timeWeightedAmount = (mintCostLP * timeWeightBps) / BPS

Step 2 — Pro-Rata with 20% Reserve:
  vaultProRata = (currentBalance * 8000 * mintCostLP) / (BPS * totalActiveMintCost)

Step 3 — Gross Redemption:
  grossRedemption = min(timeWeightedAmount, vaultProRata)

Step 4 — Sliding Fee (20% -> 3% over 365 days):
  feeBps = max(300, 2000 - (1700 * daysSinceMint) / DAYS_365)
  fee = (grossRedemption * feeBps) / BPS

Step 5 — Net and Fee Split:
  lpReturned = grossRedemption - fee
  feeBurned = fee / 2           -> sent to 0xdead
  feeToVault = fee - feeBurned  -> stays in vault
```

**Reverts**:
- No mint record exists for token
- Cooldown not elapsed: `block.timestamp < lastRedemptionTime[holder] + redemptionCooldown`

**Effects**: Deletes `mintRecords[tokenId]`, decrements `totalActiveMintCost`, updates `lastRedemptionTime[holder]`.

### `previewRedemption(uint256 tokenId) -> (grossAmount, fee, netAmount, timeWeightBps, feeBps)`

Non-mutating preview of redemption values.

```solidity
function previewRedemption(uint256 tokenId) external view returns (
```

## Harvest (Permissionless)

### `harvest()`

Distributes vault surplus above target. Anyone can call after `harvestMinInterval`.

```solidity
function harvest() external nonReentrant
```

**Process**:

```
1. Check: block.timestamp >= lastHarvestTime + harvestMinInterval
2. Calculate target: vaultTarget = (3000 * totalActiveMintCost) / BPS
3. If currentBalance <= target: set lastHarvestTime, return (no-op)
4. harvestable = currentBalance - target
5. Select tier based on vault health:
   - health = (currentBalance * BPS) / target
   - >= 8000: Healthy tier
   - >= 5000: Medium tier
   - < 5000: Low tier
6. callerBounty = min(harvestable * bountyBps / BPS, bountyCap)
7. distributable = harvestable - callerBounty
8. dsfoBonusAmount = distributable * dsfoBonusBps / BPS -> sent to FeeManager
9. treasuryAmount = distributable * treasuryBps / BPS -> sent to treasury
10. retained = distributable - dsfoBonusAmount - treasuryAmount -> stays in vault
11. Transfer bounty to caller
```

The DSFO bonus LP sent to FeeManager is broken down and distributed to holders during the next trigger cycle.
