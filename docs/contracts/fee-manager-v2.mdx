# FeeManagerV2

**Address**: [`0x852a62798B691eB5d8424169797b25e786DF4911`](https://peaq.subscan.io/account/0x852a62798B691eB5d8424169797b25e786DF4911?tab=contract)

Breaks down LP tokens into underlying assets and accumulates rewards per share. Holders call `claimAllFees()` to withdraw — no gas-heavy distribution loops. Gas cost of triggering is O(LP_tokens) instead of O(LP_tokens x holders).

**Inherits**: Ownable, ReentrancyGuard

**Source**: `contracts/FeeManager_v2.sol`

## Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `PRECISION` | 1e36 | Reward accounting precision — prevents truncation with small amounts and many holders |
| `MAX_REWARD_TOKENS` | 50 | Maximum unique reward token types that can be tracked |

## Core Functions

### `triggerBreakdownAndDistribution(uint256[] minAmounts0, uint256[] minAmounts1)`

Permissionless. Breaks all registered LP tokens into underlying assets with slippage protection.

```solidity
function triggerBreakdownAndDistribution(
    uint256[] calldata minAmounts0,
    uint256[] calldata minAmounts1
) external nonReentrant
```

**Parameters**:
- `minAmounts0` — Minimum token0 output per LP token (slippage protection). Must match `lpTokenAddresses` length.
- `minAmounts1` — Minimum token1 output per LP token. Must match `lpTokenAddresses` length.

**Process**:
1. Validates `block.timestamp >= lastTriggerTime + triggerMinInterval`
2. If `totalTrackedShares == 0`: sends all LP to treasury (fallback)
3. For each registered LP token with balance >= `triggerMinLPBalance`:
   - Approves LP to router
   - Calls `router.removeLiquidity(token0, token1, lpBalance, minAmount0, minAmount1, address(this), deadline)`
   - Calculates caller bounty: `bounty = min(amount * bountyBps / 10000, bountyCap)`
   - Transfers bounty to `msg.sender`
   - Accumulates remainder: `accRewardPerShare[token] += (remainder * 1e36) / totalTrackedShares`
4. Sets `lastTriggerTime = block.timestamp`

**Reverts**:
- `"Too soon"` if minimum interval not elapsed
- `removeLiquidity` revert if slippage bounds exceeded

Backwards-compatible overload with zero slippage protection.

### `claimAllFees()`

Claim all pending fees across all reward tokens.

```solidity
function claimAllFees() external nonReentrant
```

**Process**:
1. Settles accrued rewards for caller (moves newly accrued to `pendingRewards`)
2. For each reward token with pending balance > 0: transfers to caller
3. Updates `rewardDebt` for all tokens

### `claimFees(address[] calldata tokens)`

Claim pending fees for specific tokens only.

```solidity
function claimFees(address[] calldata tokens) external nonReentrant
```

### `handleOwnershipChange(address from, address to, uint256 amount)`

Called by DSFO token on every mint/burn to update holder shares.

```solidity
function handleOwnershipChange(address from, address to, uint256 amount) external onlyDsfoToken
```

**Modifier**: `onlyDsfoToken` — only callable by the registered DSFO contract.

**Process** (for each non-zero address):
1. Settle existing rewards into `pendingRewards`
2. Update holder balance (increase for `to`, decrease for `from`)
3. Add/remove from `holderList` as needed (swap-and-pop)
4. Update `totalTrackedShares`
5. Recalculate `rewardDebt`

Skips if `from == to` or `amount == 0`.

## LP Token Management

### `addLPTokenAddress(address lpToken)` — onlyOwner

Register an LP token for breakdown. Only registered LPs are processed during trigger.

**Reverts**: If already registered or zero address.

### `registerLP(address lpToken)` — permissionless

Register an LP token after validating it's a real V2 pair from the Factory:

```solidity
function registerLP(address lpToken) external {
    require(!isRegisteredLP[lpToken], "Already registered");
    address token0 = IUniswapV2Pair(lpToken).token0();
    address token1 = IUniswapV2Pair(lpToken).token1();
    require(factory.getPair(token0, token1) == lpToken, "Not a valid Factory pair");
    // ... register
}
```

This allows anyone to register legitimate trading pairs without waiting for an owner transaction.

### `removeLPTokenAddress(uint256 index)` — onlyOwner

Remove an LP token from the registry using swap-and-pop deletion.

## Accumulator Internals

### Reward Settlement (`_settleRewards`)

Called before any balance change to snapshot accrued rewards:

```
For each rewardToken:
  accrued = (holderBalance * accRewardPerShare[token]) / PRECISION
  if accrued > rewardDebt[holder][token]:
    pendingRewards[holder][token] += accrued - debt
```

### Debt Update (`_updateRewardDebt`)

Called after balance changes to set the new baseline:

```
For each rewardToken:
  rewardDebt[holder][token] = (newBalance * accRewardPerShare[token]) / PRECISION
```

### Holder Tracking (Swap-and-Pop)

Holders are stored in an array with O(1) removal using swap-and-pop. `holderIndexPlusOne` maps addresses to their array position (+1, so 0 means "not in list").
