# FeeManagerV2

**Address**: `0x852a62798B691eB5d8424169797b25e786DF4911`

Breaks down LP tokens into underlying assets and accumulates rewards per share. Holders call `claimAllFees()` to withdraw — no gas-heavy distribution loops. Gas cost of triggering is O(LP_tokens) instead of O(LP_tokens x holders).

**Inherits**: Ownable, ReentrancyGuard

## Core Functions

### `triggerBreakdownAndDistribution(uint256[] minAmounts0, uint256[] minAmounts1)`

Permissionless. Breaks all registered LP tokens into underlying assets with slippage protection.

**Process**:
1. Validates minimum interval has passed
2. If no holders: sends LP to treasury (fallback)
3. For each registered LP token with sufficient balance:
   - Calls `router.removeLiquidity()` with min amounts
   - Deducts caller bounty from each output token
   - Accumulates remaining into `accRewardPerShare`

**Slippage arrays**: Must match `lpTokenAddresses` length. Pass 0 for tokens you want unprotected.

### `triggerBreakdownAndDistribution()` (no args)

Same as above but with 0 slippage protection (backwards-compatible overload).

### `claimAllFees()`

Claim all pending fees across all reward tokens. Settles accrued rewards, transfers each token with a non-zero balance.

### `claimFees(address[] tokens)`

Claim pending fees for specific tokens only.

### `handleOwnershipChange(address from, address to, uint256 amount)`

**Callable only by DSFO token contract.** Called on every mint/burn to update holder shares.

For both `from` and `to` (when non-zero):
1. Settle existing rewards into `pendingRewards`
2. Adjust holder balance (increase/decrease)
3. Update reward debt

## LP Token Management

### `addLPTokenAddress(address)` (onlyOwner)

Register an LP token for breakdown. Only registered LPs are processed during trigger.

### `registerLP(address)` (permissionless)

Register an LP token after validating it's a real V2 pair from the Factory:

```solidity
require(factory.getPair(token0, token1) == lpToken, "Not a valid Factory pair");
```

### `removeLPTokenAddress(uint256 index)` (onlyOwner)

Remove an LP token from the registry (swap-and-pop).

## View Functions

### `claimableAll(address holder) → (address[] tokens, uint256[] amounts)`

Returns claimable amount for a holder across all reward tokens. Used by the frontend dashboard.

### `claimable(address holder, address token) → uint256`

Returns claimable amount for a specific holder + token pair.
